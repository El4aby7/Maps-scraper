# General Workflow

- Think before making changes.
- Create a plan for multi-step tasks.
- Explain tradeoffs when there are multiple solutions.
- Prefer minimal changes over large rewrites.

# Context Navigation

- Query Graphify before reading files.
- Use graphify-out/wiki/index.md as the starting point.
- Read raw files only when necessary.

# Coding

- Follow existing project conventions.
- Avoid introducing new dependencies without justification.
- Update documentation when behavior changes.

# Communication

- Be concise.
- Surface risks early.
- State assumptions explicitly.
